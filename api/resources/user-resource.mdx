The User resource represents a user account in the Ash Coffee application. It includes authentication, email confirmation, and password management capabilities.

## Resource Overview

<ParamField path="type" type="string">
  Resource type: `"user"`
</ParamField>

<ParamField path="id" type="string">
  UUID primary key
</ParamField>

## Attributes

### id

<ResponseField name="id" type="uuid" required>
  Unique identifier for the user (UUID v4)
</ResponseField>

```json
"id": "123e4567-e89b-12d3-a456-426614174000"
```

### email

<ResponseField name="email" type="ci_string" required>
  User's email address. Case-insensitive string that serves as the primary identity field.
</ResponseField>

**Constraints:**
- Must be unique
- Case-insensitive
- Required for registration
- Cannot be null

```json
"email": "user@example.com"
```

### hashed_password

<ResponseField name="hashed_password" type="string" sensitive>
  Bcrypt-hashed password. This field is sensitive and never returned in API responses.
</ResponseField>

**Security:**
- Hashed using bcrypt
- Never exposed in API responses
- Required for password authentication
- Minimum 8 characters (before hashing)

<Warning>
  This field is never returned in API responses for security reasons.
</Warning>

### confirmed_at

<ResponseField name="confirmed_at" type="utc_datetime_usec">
  Timestamp when the user's email was confirmed. Null if email is not yet confirmed.
</ResponseField>

```json
"confirmed_at": "2026-03-03T10:30:00.123456Z"
```

or

```json
"confirmed_at": null
```

## Actions

The User resource supports the following actions:

### read (Default)

<ParamField path="action" type="string" default="read">
  Standard read action for retrieving user records
</ParamField>

**Usage:**
```bash
GET /api/json/users/{id}
```

### register_with_password (Create)

<ParamField path="action" type="string">
  `register_with_password` - Create a new user account
</ParamField>

**Arguments:**
- `email` (ci_string, required) - User's email address
- `password` (string, required, sensitive) - Password (min 8 characters)
- `password_confirmation` (string, required, sensitive) - Must match password

**Returns:**
- User resource
- JWT token in metadata

**Side Effects:**
- Sends confirmation email
- Generates authentication token
- Hashes password with bcrypt

### sign_in_with_password (Read)

<ParamField path="action" type="string">
  `sign_in_with_password` - Authenticate with email and password
</ParamField>

**Arguments:**
- `email` (ci_string, required) - User's email
- `password` (string, required, sensitive) - User's password

**Returns:**
- User resource (if authentication succeeds)
- JWT token in metadata

### sign_in_with_token (Read)

<ParamField path="action" type="string">
  `sign_in_with_token` - Exchange a short-lived sign-in token for a standard token
</ParamField>

**Arguments:**
- `token` (string, required, sensitive) - Short-lived sign-in token

**Returns:**
- User resource
- Standard JWT token in metadata

### get_by_email (Read)

<ParamField path="action" type="string">
  `get_by_email` - Look up a user by email address
</ParamField>

**Arguments:**
- `email` (ci_string, required) - Email to search for

**Returns:**
- User resource (if found)

### get_by_subject (Read)

<ParamField path="action" type="string">
  `get_by_subject` - Retrieve user by JWT subject claim
</ParamField>

**Arguments:**
- `subject` (string, required) - Subject claim from JWT (user ID)

**Returns:**
- User resource (if found)

### change_password (Update)

<ParamField path="action" type="string">
  `change_password` - Update user's password
</ParamField>

**Arguments:**
- `current_password` (string, required, sensitive) - Current password for verification
- `password` (string, required, sensitive) - New password (min 8 characters)
- `password_confirmation` (string, required, sensitive) - Must match new password

**Side Effects:**
- Revokes all existing tokens (log out everywhere)
- Hashes new password with bcrypt

<Warning>
  Changing a password revokes ALL existing authentication tokens for security.
</Warning>

### request_password_reset_token (Action)

<ParamField path="action" type="string">
  `request_password_reset_token` - Request a password reset
</ParamField>

**Arguments:**
- `email` (ci_string, required) - Email of account to reset

**Side Effects:**
- Sends password reset email if account exists
- Creates a time-limited reset token

**Security:**
- Always returns success (even if email doesn't exist)
- Prevents email enumeration attacks

### reset_password_with_token (Update)

<ParamField path="action" type="string">
  `reset_password_with_token` - Reset password using a reset token
</ParamField>

**Arguments:**
- `reset_token` (string, required, sensitive) - Token from password reset email
- `password` (string, required, sensitive) - New password (min 8 characters)
- `password_confirmation` (string, required, sensitive) - Must match new password

**Returns:**
- User resource
- New JWT token in metadata

**Side Effects:**
- Invalidates reset token
- Generates new authentication token
- Auto-confirms email if not already confirmed

## Identities

### unique_email

<ParamField path="identity" type="string">
  `unique_email` - Ensures email uniqueness across all users
</ParamField>

**Fields:** `[:email]`

This identity constraint ensures that each email can only be registered once in the system.

## Authentication Configuration

The User resource is configured with several authentication strategies:

### Password Strategy

<Expandable title="Password Authentication Details">
  - **Identity Field:** email
  - **Hash Provider:** bcrypt
  - **Resettable:** Yes (via email)
  - **Password Constraints:** Minimum 8 characters
  - **Confirmation Required:** Yes (must match password_confirmation)
</Expandable>

### Remember Me Strategy

<Expandable title="Remember Me Details">
  Allows users to maintain longer-lived sessions without re-authenticating.
</Expandable>

### Magic Link Strategy

<Expandable title="Magic Link Details">
  Passwordless authentication via email link.
</Expandable>

### Confirmation Add-on

<Expandable title="Email Confirmation Details">
  - **Monitor Fields:** `[:email]`
  - **Confirm on Create:** Yes
  - **Confirm on Update:** No
  - **Require Interaction:** Yes
  - **Auto-confirm Actions:** sign_in_with_magic_link, reset_password_with_token
  - **Confirmed At Field:** confirmed_at
</Expandable>

### Log Out Everywhere Add-on

<Expandable title="Log Out Everywhere Details">
  - **Apply on Password Change:** Yes
  - Revokes all tokens when password is changed
</Expandable>

## Token Configuration

<ParamField path="tokens.enabled" type="boolean" default="true">
  JWT tokens are enabled for this resource
</ParamField>

<ParamField path="tokens.token_resource" type="resource">
  Uses `AshCoffee.Accounts.Token` for token storage
</ParamField>

<ParamField path="tokens.store_all_tokens" type="boolean" default="true">
  All tokens are stored in the database
</ParamField>

<ParamField path="tokens.require_token_presence_for_authentication" type="boolean" default="true">
  Tokens must exist in database to be valid
</ParamField>
