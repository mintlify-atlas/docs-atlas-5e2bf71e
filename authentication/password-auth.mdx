## Overview

Ash Coffee uses password-based authentication with bcrypt hashing, email confirmation, and password reset capabilities. All password operations are defined as actions on the User resource.

## Registration

Users can register with an email and password using the `register_with_password` action.

### Registration Action

```elixir lib/ash_coffee/accounts/user.ex
create :register_with_password do
  description "Register a new user with a email and password."

  argument :email, :ci_string do
    allow_nil? false
  end

  argument :password, :string do
    description "The proposed password for the user, in plain text."
    allow_nil? false
    constraints min_length: 8
    sensitive? true
  end

  argument :password_confirmation, :string do
    description "The proposed password for the user (again), in plain text."
    allow_nil? false
    sensitive? true
  end

  # Sets the email from the argument
  change set_attribute(:email, arg(:email))

  # Hashes the provided password
  change AshAuthentication.Strategy.Password.HashPasswordChange

  # Generates an authentication token for the user
  change AshAuthentication.GenerateTokenChange

  # validates that the password matches the confirmation
  validate AshAuthentication.Strategy.Password.PasswordConfirmationValidation

  metadata :token, :string do
    description "A JWT that can be used to authenticate the user."
    allow_nil? false
  end
end
```

### Usage Example

<CodeGroup>
```elixir Elixir
# Register a new user
case AshCoffee.Accounts.User
     |> Ash.Changeset.for_create(:register_with_password, %{
       email: "user@example.com",
       password: "secure_password_123",
       password_confirmation: "secure_password_123"
     })
     |> Ash.create() do
  {:ok, user} ->
    # Registration successful
    token = user.__metadata__.token
    {:ok, user, token}

  {:error, changeset} ->
    # Handle validation errors
    {:error, changeset}
end
```

```elixir Phoenix Controller
def register(conn, %{"user" => user_params}) do
  case AshCoffee.Accounts.User
       |> Ash.Changeset.for_create(:register_with_password, user_params)
       |> Ash.create() do
    {:ok, user} ->
      token = user.__metadata__.token
      
      conn
      |> put_status(:created)
      |> json(%{user: user, token: token})

    {:error, changeset} ->
      conn
      |> put_status(:unprocessable_entity)
      |> json(%{errors: changeset.errors})
  end
end
```
</CodeGroup>

<Note>
The registration action automatically:
- Hashes the password with bcrypt
- Validates password confirmation matches
- Generates a JWT token for immediate authentication
- Triggers email confirmation workflow
</Note>

## Sign In

Users sign in with their email and password using the `sign_in_with_password` action.

### Sign In Action

```elixir lib/ash_coffee/accounts/user.ex
read :sign_in_with_password do
  description "Attempt to sign in using a email and password."
  get? true

  argument :email, :ci_string do
    description "The email to use for retrieving the user."
    allow_nil? false
  end

  argument :password, :string do
    description "The password to check for the matching user."
    allow_nil? false
    sensitive? true
  end

  # validates the provided email and password and generates a token
  prepare AshAuthentication.Strategy.Password.SignInPreparation

  metadata :token, :string do
    description "A JWT that can be used to authenticate the user."
    allow_nil? false
  end
end
```

### Usage Example

<CodeGroup>
```elixir Elixir
# Sign in a user
case AshCoffee.Accounts.User
     |> Ash.Query.for_read(:sign_in_with_password, %{
       email: "user@example.com",
       password: "secure_password_123"
     })
     |> Ash.read_one() do
  {:ok, user} ->
    # Sign in successful
    token = user.__metadata__.token
    {:ok, user, token}

  {:error, _error} ->
    # Invalid credentials
    {:error, :invalid_credentials}
end
```

```elixir Phoenix Controller
def sign_in(conn, %{"email" => email, "password" => password}) do
  case AshCoffee.Accounts.User
       |> Ash.Query.for_read(:sign_in_with_password, %{
         email: email,
         password: password
       })
       |> Ash.read_one() do
    {:ok, user} ->
      token = user.__metadata__.token
      
      conn
      |> json(%{user: user, token: token})

    {:error, _error} ->
      conn
      |> put_status(:unauthorized)
      |> json(%{error: "Invalid email or password"})
  end
end
```
</CodeGroup>

## Password Reset

Users can reset their password through a two-step process: requesting a reset token and using it to set a new password.

### Step 1: Request Password Reset

```elixir lib/ash_coffee/accounts/user.ex
action :request_password_reset_token do
  description "Send password reset instructions to a user if they exist."

  argument :email, :ci_string do
    allow_nil? false
  end

  # creates a reset token and invokes the relevant senders
  run {AshAuthentication.Strategy.Password.RequestPasswordReset, action: :get_by_email}
end
```

<CodeGroup>
```elixir Request Reset Token
# Request a password reset email
AshCoffee.Accounts.User
|> Ash.ActionInput.for_action(:request_password_reset_token, %{
  email: "user@example.com"
})
|> Ash.run_action()

# Always returns :ok to prevent email enumeration
{:ok, nil}
```

```elixir Phoenix Controller
def request_password_reset(conn, %{"email" => email}) do
  # Always return success to prevent email enumeration
  AshCoffee.Accounts.User
  |> Ash.ActionInput.for_action(:request_password_reset_token, %{email: email})
  |> Ash.run_action()

  conn
  |> json(%{message: "If an account exists, a password reset email has been sent"})
end
```
</CodeGroup>

<Warning>
The password reset request always returns success, even if the email doesn't exist. This prevents attackers from enumerating valid email addresses.
</Warning>

### Step 2: Reset Password with Token

```elixir lib/ash_coffee/accounts/user.ex
update :reset_password_with_token do
  argument :reset_token, :string do
    allow_nil? false
    sensitive? true
  end

  argument :password, :string do
    description "The proposed password for the user, in plain text."
    allow_nil? false
    constraints min_length: 8
    sensitive? true
  end

  argument :password_confirmation, :string do
    description "The proposed password for the user (again), in plain text."
    allow_nil? false
    sensitive? true
  end

  # validates the provided reset token
  validate AshAuthentication.Strategy.Password.ResetTokenValidation

  # validates that the password matches the confirmation
  validate AshAuthentication.Strategy.Password.PasswordConfirmationValidation

  # Hashes the provided password
  change AshAuthentication.Strategy.Password.HashPasswordChange

  # Generates an authentication token for the user
  change AshAuthentication.GenerateTokenChange
end
```

```

```
</CodeGroup>

<Note>
Resetting a password triggers the "log out everywhere" feature, invalidating all existing tokens for the user.
</Note>

## Change Password

Authenticated users can change their password by providing their current password.

```elixir lib/ash_coffee/accounts/user.ex
update :change_password do
  require_atomic? false
  accept []
  
  argument :current_password, :string, sensitive?: true, allow_nil?: false
  argument :password, :string, sensitive?: true, allow_nil?: false, constraints: [min_length: 8]
  argument :password_confirmation, :string, sensitive?: true, allow_nil?: false

  validate confirm(:password, :password_confirmation)
  validate {AshAuthentication.Strategy.Password.PasswordValidation, strategy_name: :password, password_argument: :current_password}
  change {AshAuthentication.Strategy.Password.HashPasswordChange, strategy_name: :password}
end
```

### Usage Example

```elixir
# Change password for authenticated user
case user
     |> Ash.Changeset.for_update(:change_password, %{
       current_password: "old_password_123",
       password: "new_password_456",
       password_confirmation: "new_password_456"
     })
     |> Ash.update() do
  {:ok, user} ->
    # Password changed successfully
    # All existing tokens are revoked
    {:ok, user}

  {:error, changeset} ->
    # Invalid current password or validation errors
    {:error, changeset}
end
```

<Warning>
Changing a password automatically triggers "log out everywhere", revoking all active sessions. The user will need to sign in again on all devices.
</Warning>

## Remember Me

The remember me strategy extends session duration for persistent authentication.

```elixir lib/ash_coffee/accounts/user.ex
authentication do
  strategies do
    password :password do
      # ... password config
    end

    remember_me :remember_me
  end
end
```

<Note>
The remember me strategy is automatically configured and works alongside password authentication to provide extended session tokens.
</Note>

## Password Requirements

  Minimum length: 8 characters
  
  Marked as sensitive to prevent logging
</ParamField>

<ParamField path="password_confirmation" type="string" required>
  Must match the password exactly
  
  Validates user intent and prevents typos
</ParamField>

## Best Practices

<Steps>
  <Step title="Use Strong Passwords">
    Enforce minimum 8 characters (already configured). Consider implementing additional complexity requirements in your UI.
  </Step>
  
  <Step title="Handle Errors Gracefully">
    Don't expose whether an email exists during password reset to prevent enumeration attacks.
  </Step>
  
  <Step title="Store Tokens Securely">
    Store JWT tokens in HTTP-only cookies or secure storage, never in localStorage for production apps.
  </Step>
  
  <Step title="Implement Rate Limiting">
    Add rate limiting to sign-in and password reset endpoints to prevent brute force attacks.
  </Step>
</Steps>
