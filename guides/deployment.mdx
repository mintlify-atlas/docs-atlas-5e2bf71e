This guide covers deploying your Ash Coffee application to production.

## Pre-Deployment Checklist

<Steps>
  <Step title="Review configuration">
    Ensure all required environment variables are set:
    - `DATABASE_URL`
    - `SECRET_KEY_BASE`
    - `TOKEN_SIGNING_SECRET`
    - `PHX_HOST`
  </Step>
  
  <Step title="Run pre-commit checks">
    ```bash
    mix precommit
    ```
    
    This runs:
    - Compile with warnings as errors
    - Remove unused dependencies
    - Format code
    - Run tests
  </Step>
  
  <Step title="Set production config">
    Review `config/prod.exs` and `config/runtime.exs` for production settings.
  </Step>
  
  <Step title="Prepare assets">
    Build and optimize assets for production:
    
    ```bash
    mix assets.deploy
    ```
  </Step>
</Steps>

## Environment Variables

Create a `.env.production` file with all required variables:

```bash .env.production
# Database
DATABASE_URL=ecto://user:password@db.example.com:5432/ash_coffee_prod
POOL_SIZE=10

# Application
SECRET_KEY_BASE=your_secret_key_base_here
TOKEN_SIGNING_SECRET=your_token_signing_secret_here
PHX_HOST=example.com
PORT=4000

# Server
PHX_SERVER=true

# Optional
ECTO_IPV6=false
DNS_CLUSTER_QUERY=

# Mailer (choose one adapter)
MAILGUN_API_KEY=your_mailgun_key
MAILGUN_DOMAIN=mg.example.com
# or
SENDGRID_API_KEY=your_sendgrid_key
# or
SMTP_RELAY=smtp.example.com
SMTP_USERNAME=your_username
SMTP_PASSWORD=your_password
```

<Warning>
Never commit `.env.production` to version control. Add it to `.gitignore`.
</Warning>

## Asset Compilation

The `mix assets.deploy` alias handles all asset compilation and optimization:

```elixir mix.exs
aliases do
  [
    "assets.deploy": [
      "tailwind ash_coffee --minify",
      "esbuild ash_coffee --minify",
      "phx.digest"
    ]
  ]
end
```

This command:
1. Compiles and minifies Tailwind CSS
2. Bundles and minifies JavaScript with esbuild
3. Generates digested static assets with cache busting

### Manual Asset Steps

If you need to run steps individually:

<CodeGroup>
```bash Tailwind CSS
mix tailwind ash_coffee --minify
```

```bash JavaScript
mix esbuild ash_coffee --minify
```

```bash Digest Assets
mix phx.digest
```
</CodeGroup>

## Building a Release

Ash Coffee can be deployed as a Mix release for better performance and smaller deployments.

<Steps>
  <Step title="Set MIX_ENV to production">
    ```bash
    export MIX_ENV=prod
    ```
  </Step>
  
  <Step title="Install dependencies">
    ```bash
    mix deps.get --only prod
    ```
  </Step>
  
  <Step title="Compile the application">
    ```bash
    mix compile
    ```
  </Step>
  
  <Step title="Compile assets">
    ```bash
    mix assets.deploy
    ```
  </Step>
  
  <Step title="Build the release">
    ```bash
    mix release
    ```
    
    The release will be created in `_build/prod/rel/ash_coffee/`.
  </Step>
</Steps>

### Running the Release

```bash
# Set environment variables
export DATABASE_URL="ecto://user:pass@host/db"
export SECRET_KEY_BASE="..."
export TOKEN_SIGNING_SECRET="..."
export PHX_HOST="example.com"
export PHX_SERVER=true

# Start the server
_build/prod/rel/ash_coffee/bin/ash_coffee start
```

<Note>
The `PHX_SERVER=true` environment variable is required to start the web server in a release.
</Note>

## Database Migrations

Run migrations before starting the application:

```bash
mix ecto.migrate
```

For releases, you can run migrations in the application startup:

```elixir lib/ash_coffee/application.ex
defmodule AshCoffee.Application do
  def start(_type, _args) do
    # Run migrations on startup
    AshCoffee.Release.migrate()
    
    children = [
      # ...
    ]
    
    Supervisor.start_link(children, strategy: :one_for_one, name: AshCoffee.Supervisor)
  end
end
```

Create a release module:

```elixir lib/ash_coffee/release.ex
defmodule AshCoffee.Release do
  @moduledoc """
  Tasks for running in releases.
  """
  @app :ash_coffee

  def migrate do
    load_app()

    for repo <- repos() do
      {:ok, _, _} = Ecto.Migrator.with_repo(repo, &Ecto.Migrator.run(&1, :up, all: true))
    end
  end

  def rollback(repo, version) do
    load_app()
    {:ok, _, _} = Ecto.Migrator.with_repo(repo, &Ecto.Migrator.run(&1, :down, to: version))
  end

  defp repos do
    Application.fetch_env!(@app, :ecto_repos)
  end

  defp load_app do
    Application.load(@app)
  end
end
```

## Deployment Platforms

### Fly.io

<Steps>
  <Step title="Install flyctl">
    ```bash
    curl -L https://fly.io/install.sh | sh
    ```

  </Step>

  <Step title="Create a PostgreSQL database">

  <Step title="Deploy">
    ```bash
    fly deploy
    ```

</Steps>

### Render

<Steps>
  <Step title="Create a new Web Service">
    Connect your GitHub repository to Render.
  </Step>
  
  <Step title="Configure build settings">
    **Build Command:**
    ```bash
    mix deps.get && mix assets.deploy && mix compile
    ```
    
    **Start Command:**
    ```bash
    mix phx.server
    ```
  </Step>
  
  <Step title="Add environment variables">

  <Step title="Create PostgreSQL database">
    Create a PostgreSQL database service in Render and link it to your web service.
  </Step>
  
  <Step title="Deploy">
    Render will automatically deploy on git push.
  </Step>
</Steps>

## Health Checks

Implement a health check endpoint for monitoring:

```elixir lib/ash_coffee_web/controllers/health_controller.ex
defmodule AshCoffeeWeb.HealthController do
  use AshCoffeeWeb, :controller

  def show(conn, _params) do
    # Check database connection
    case Ecto.Adapters.SQL.query(AshCoffee.Repo, "SELECT 1") do
      {:ok, _} ->
        json(conn, %{status: "healthy"})
      {:error, _} ->
        conn
        |> put_status(:service_unavailable)
        |> json(%{status: "unhealthy"})
    end
  end
end
```

Add the route:

```elixir lib/ash_coffee_web/router.ex
scope "/" do
  pipe_through :api
  get "/health", AshCoffeeWeb.HealthController, :show
end
```

## SSL/TLS Configuration

### Force SSL

SSL is enforced by default in production:

```elixir config/prod.exs
config :ash_coffee, AshCoffeeWeb.Endpoint,
  force_ssl: [rewrite_on: [:x_forwarded_proto]],
  exclude: [
    hosts: ["localhost", "127.0.0.1"]
  ]
```

### Custom Certificates

For custom SSL certificates:

```elixir config/runtime.exs
config :ash_coffee, AshCoffeeWeb.Endpoint,
  https: [
    port: 443,
    cipher_suite: :strong,
    keyfile: System.get_env("SSL_KEY_PATH"),
    certfile: System.get_env("SSL_CERT_PATH")
  ]
```

## Performance Optimization

### Database Connection Pooling

Adjust pool size based on your load:

```bash
POOL_SIZE=20  # For production with high traffic
```

### Asset Compression

Gzip is handled by the CDN or reverse proxy. For nginx:

```nginx
gzip on;
gzip_vary on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
```

### Caching

Static assets are automatically cached with digest filenames:

```elixir config/prod.exs
config :ash_coffee, AshCoffeeWeb.Endpoint,
  cache_static_manifest: "priv/static/cache_manifest.json"
```

## Troubleshooting

### Application Won't Start

1. Check environment variables are set:
   ```bash
   printenv | grep -E '(DATABASE_URL|SECRET_KEY_BASE|TOKEN_SIGNING_SECRET)'
   ```

2. Verify database connectivity:
   ```bash
   psql $DATABASE_URL
   ```

3. Check logs for errors:
   ```bash
   # For releases
   tail -f _build/prod/rel/ash_coffee/log/erlang.log.1
   
   # For Docker
   docker-compose logs -f app
   ```

### Assets Not Loading

1. Ensure `mix assets.deploy` was run
2. Check `priv/static/cache_manifest.json` exists
3. Verify static file serving is enabled

### Database Connection Issues

1. Verify `DATABASE_URL` format:
   ```
   ecto://username:password@host:port/database
   ```

2. Check SSL requirements:
   ```elixir
   config :ash_coffee, AshCoffee.Repo,
     ssl: true,
     ssl_opts: [verify: :verify_none]
   ```

3. Test connection pool size - increase if seeing timeout errors
