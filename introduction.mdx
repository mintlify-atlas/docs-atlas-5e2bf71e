# Welcome to Ash Coffee

Ash Coffee is a production-ready Phoenix web application built with the powerful [Ash Framework](https://ash-hq.org/). It provides a complete foundation for building secure, scalable web applications with built-in user authentication, policy-based authorization, and a JSON API.

## What is Ash Coffee?

Ash Coffee demonstrates best practices for building Phoenix applications using Ash Framework's declarative resource definitions, eliminating boilerplate while maintaining flexibility and type safety. Built on Phoenix 1.8+ with LiveView, it offers a modern development experience with real-time features and excellent developer ergonomics.

## Architecture Overview

Ash Coffee follows a domain-driven design pattern powered by the Ash Framework:

- **Domains**: Logical groupings of resources (e.g., `AshCoffee.Accounts`)
- **Resources**: Declarative definitions of data models with actions, policies, and validations
- **Actions**: Type-safe operations (read, create, update, destroy) with built-in changesets
- **Policies**: Authorization rules that run automatically on every action

### Core Components

**Authentication System**
- User registration with email confirmation
- Password authentication with secure bcrypt hashing
- Password reset via email tokens
- Remember me functionality for persistent sessions
- Magic link authentication support
- JWT token management for API access

**API Layer**
- JSON API compliant endpoints at `/api/json`
- Interactive Swagger UI at `/api/json/swaggerui`
- Automatic OpenAPI specification generation
- Bearer token authentication for API requests

**Development Tools**
- Phoenix LiveDashboard for monitoring at `/dev/dashboard`
- Email preview with Swoosh at `/dev/mailbox`
- Hot code reloading for rapid development
- Comprehensive test suite with ExUnit
